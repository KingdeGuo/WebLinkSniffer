# 域名过滤配置功能实现总结

## 功能概述
新增了配置页面功能，允许用户自定义过滤特定域名的链接。默认过滤推特、Facebook、Instagram等社交媒体域名。

## 实现的功能

### 1. 配置页面（options.html + options.js）
- **界面设计**：现代化的渐变背景和卡片式布局
- **过滤设置**：
  - 启用/禁用域名过滤
  - 选择是否隐藏被过滤的链接
- **预设域名**：一键添加常用社交媒体域名
  - Twitter、X (Twitter)、Facebook、Instagram、TikTok、微博、抖音、B站
- **自定义域名**：
  - 手动添加单个域名
  - 批量添加多个域名（每行一个）
  - 删除已添加的域名
- **批量操作**：
  - 批量添加域名
  - 清空所有域名
  - 恢复默认设置

### 2. 内容脚本增强（content.js）
- 新增 `getFilteredLinks()` 异步函数
- 从存储中获取过滤配置
- 根据配置过滤链接：
  - 支持子域名匹配（如 `api.twitter.com` 也会被 `twitter.com` 过滤）
  - 可选择隐藏或显示被过滤的链接

### 3. 弹出窗口增强（popup.html + popup.js）
- 添加"配置"按钮（⚙️ 图标）
- 点击配置按钮打开配置页面

### 4. 清单文件更新（manifest.json）
- 添加 `options_ui` 配置
- 版本号更新为 1.1

## 默认过滤域名
```javascript
const DEFAULT_FILTERED_DOMAINS = [
    'twitter.com',
    'x.com',
    'facebook.com',
    'instagram.com',
    'tiktok.com',
    'weibo.com',
    'douyin.com'
];
```

## 使用方法

### 访问配置页面
1. 点击浏览器工具栏中的插件图标
2. 在弹出窗口中点击"⚙️ 配置"按钮
3. 配置页面将在新标签页中打开

### 添加过滤域名
1. **预设域名**：点击预设标签快速添加
2. **手动添加**：在输入框中输入域名，点击"添加域名"
3. **批量添加**：在文本框中每行输入一个域名，点击"批量添加"

### 域名格式支持
- 简单域名：`example.com`
- 带协议：`https://example.com`
- 带路径：`https://example.com/path`
- 带端口：`example.com:8080`
- 子域名自动匹配：添加 `twitter.com` 后，`api.twitter.com` 也会被过滤

## 技术细节

### 存储结构
```javascript
{
    filteredDomains: ['twitter.com', 'facebook.com', ...],
    enableFilter: true,
    hideFiltered: true
}
```

### 过滤逻辑
1. 从存储加载过滤配置
2. 解析每个链接的域名
3. 检查域名是否在过滤列表中（支持子域名匹配）
4. 根据配置决定是否隐藏被过滤的链接

### 数据持久化
- 使用 `chrome.storage.local` 存储配置
- 配置在浏览器重启后仍然有效
- 支持恢复默认设置

## 测试场景
1. 添加推特相关域名，验证链接被正确过滤
2. 切换"启用过滤"开关，验证过滤功能开关
3. 切换"隐藏被过滤链接"选项，验证显示/隐藏行为
4. 添加子域名，验证父域名过滤规则生效
5. 批量添加域名，验证批量操作功能
6. 恢复默认设置，验证默认配置恢复

## 注意事项
1. 域名匹配不区分大小写
2. 修改配置后需要刷新链接列表才能生效
3. 清空所有域名会清空包括默认域名在内的所有配置
4. 恢复默认设置会覆盖当前所有配置
