测试域名过滤功能

社交媒体链接（应该被过滤）

  • Twitter 链接
  • X (Twitter) 链接
  • Facebook 链接
  • Instagram 链接
  • TikTok 链接
  • 微博链接
  • 抖音链接

子域名测试（应该被过滤）

  • API Twitter 链接
  • Mobile Twitter 链接
  • Graph Facebook 链接

普通链接（不应该被过滤）

  • Google
  • GitHub
  • Stack Overflow
  • 百度

带#的链接（应该去重）

  • 页面1
  • 页面1#章节1
  • 页面1#章节2

测试说明：

  1. 默认情况下，社交媒体链接和子域名应该被过滤掉
  2. 普通链接应该正常显示
  3. 带#的链接应该只显示一个（不带#的）
  4. 可以在配置页面中添加或删除过滤域名
  5. 修改配置后需要刷新此页面才能看到效果

预期结果（默认配置下）：

  • 应该显示：Google、GitHub、Stack Overflow、百度、example.com/page1
  • 不应该显示：任何 twitter.com、x.com、facebook.com、instagram.com、tiktok.com、weibo.com、douyin.com 的链接
