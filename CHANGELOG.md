# 更新日志

本文档记录「网址获取」浏览器插件的所有重要变更。

## [3.0.0] - 2026-06-28

### 🧠 排序引擎 v3 — 自适应排序

- 🔄 **页面类型检测**：识别 12 种页面类型（GitHub、文档、博客、新闻、问答等），每种类型有独立的链接权重策略
- 🏷️ **链接语义分类**：content / navigation / external / resource / code / social 六种角色
- 📝 **锚文本质量评分**：分析描述性、词数、动词/名词特征，惩罚通用文本
- 📊 **会话行为追踪**：实时记录点击，识别 deep-dive / explore / browse 模式并自适应调整
- 🔥 **会话疲劳惩罚**：同域名点击过多自动降权

### 🎨 UI 全面重构

- 🎯 **极简设计**：去除所有渐变背景，采用纯白 + 浅灰的克制配色
- 📑 **分区卡片展示**：按页面区域分组（主内容、导航、侧边栏等），带图标 + 计数
- 🔘 **分组独立操作**：每个分组标题增加「全部打开」按钮
- 🟢 **重要性圆点**：绿 / 黄 / 灰三级视觉标记
- 📄 **每页 15 条**：从 5 条增加到 15 条
- 🏷️ **页面类型徽章**：顶部显示当前页面类型
- 🎭 **语义化 HTML**：header / footer / section 结构
- 🔍 **搜索框图标**：内嵌 SVG 放大镜
- 📊 **精简统计栏**：四列小号数字
- ⏩ **SVG 箭头分页**：替代文字按钮

### 🐛 Bug 修复

- 修复新链接计数不更新（打开链接后 newLinks 不递减）
- 修复 `calculateStats` 中已打开 + 已屏蔽链接的双重计数
- 修复内容区判断逻辑（遍历到 body 时默认判定为内容区）
- 修复 `enableFilter` / `enablePathPatternFilter` 开关在 popup 中未生效
- 修复屏蔽链接 / 域名后 `selectedUrls` 未清理
- 修复 `microShuffleWithinGroup` 交换后可能制造相邻重复域名
- 修复 tampermonkey 脚本 innerHTML XSS 风险
- 修复 `getFilteredLinks()` 在 `updateUI` 中被重复调用

### 📦 其他

- 新增 `ranking.js` — 自适应排序引擎
- `content.js` 返回页面元数据供页面类型检测
- manifest.json 版本号升至 3.0.0

## [2.0.0] - 2026-05-03

### 🌍 新增国际化 (i18n) 支持

#### 浏览器插件版本
- ✨ **翻译文件支持**：添加 `_locales/` 目录，包含中文(zh_CN)和英文(en)翻译
  * 50+ 个用户界面消息的完整翻译
  * 符合 Chrome Extension i18n 标准
- 🛠️ **国际化工具** (`i18n.js`)：
  * Chrome Native API 集成（`chrome.i18n.getMessage`）
  * 自动语言检测（基于浏览器 UI 语言）
  * 本地存储语言偏好设置
  * 支持带参数的动态翻译（如分页信息）
  * 优雅降级处理（翻译缺失时显示消息键）
- 📝 **HTML 集成** (`data-i18n` 属性)：
  * 简洁的模板语法
  * 自动国际化 DOM 元素
- 🎯 **JavaScript 集成**：
  * `I18n.t(key)` 获取翻译
  * `I18n.t(key, {param: value})` 参数替换
  * `I18n.setLanguage(lang)` 手动切换语言
  * `I18n.getLanguage()` 获取当前语言
- 🔧 **manifest.json 更新**：
  * 添加 `"default_locale": "en"`
  * 插件名称和描述使用 `__MSG_*__` 形式

#### 油猴脚本版本
- ✨ **内嵌国际化支持** (`tampermonkey_i18n.js`)：
  * 支持中文(zh_CN)和英文(en)
  * 完全独立，无需外部文件
  * 性能最优化
- 📦 **Tampermonkey 存储集成**：
  * 使用 `GM_setValue`/`GM_getValue` 持久化语言选择
  * 跨浏览器会话保存用户偏好
- 🌐 **浏览器语言检测**：
  * 自动检测 `navigator.language`
  * 支持多种语言变体识别
- 🎯 **API 设计**：
  * 清晰的 `TampermonkeyI18n` 类接口
  * `i18n.t(key, params)` 统一翻译方法
  * `i18n.setLanguage(lang)` 动态切换

## [1.4.0] - 2026-05-03

### 新增
- 🔀 **路径模式过滤**：支持按 URL 路径模式屏蔽链接（如 `/blog/`、`/article/`），无论域名如何，只要 URL 包含该路径模式就会被过滤
- ⚙️ **配置页面管理路径模式**：可在 options.html 中添加、删除、清空路径模式列表
- 🏷️ **路径模式快捷标签**：popup 底部展示已屏蔽的路径模式标签，支持一键撤销
- 🟢 **路径模式屏蔽按钮**：每条链接右侧新增路径模式按钮（📁），一键将当前链接的路径模式加入屏蔽列表

### 修复
- 🐛 修复路径模式（URL/域名/路径）屏蔽后，过滤未生效的问题 —— `getFilteredLinks()` 现在统一对已屏蔽链接进行实时过滤
- 🐛 修复屏蔽统计不准确的问题 —— `blockedLinks` 统计数现在反映实际被屏蔽的链接数量

## [1.3.0] - 2026-05-03

### 新增
- 🚫 链接右侧一键屏蔽按钮：单条链接直接加入屏蔽列表，无需复制粘贴到后台
- 🌐 链接右侧一键屏蔽域名按钮：一键屏蔽链接所属的整个域名
- ☑️ 多选批量操作工具栏：勾选多个链接后，支持批量屏蔽链接、批量屏蔽域名、批量复制、批量打开
- 📋 快速复制功能：选中链接后可一键复制所有选中链接的 URL 及标题
- 🔄 撤销功能：每次屏蔽/域名屏蔽后 Toast 通知带有「撤销」按钮
- 🏷️ 已屏蔽链接快捷管理：底部展示已屏蔽链接标签，点击可撤销，支持一键清空

### 优化
- 域名过滤改为始终生效（不受 options 页面开关控制），popup 一键屏蔽的域名即刻过滤
- 页面搜索功能优化
- UI 交互体验大幅提升
